## O que é um ADR?

ADR (Architecture Decision Record) é um documento curto e estruturado que registra uma decisão arquitetural importante, explicando o contexto, as opções consideradas, a decisão tomada e as consequências dela. A ideia é simples: toda decisão técnica relevante deixa um rastro escrito do "por quê", não apenas do "como".

### Critérios

Como na documentação do desafio há um item de avaliação sobre a documentação, é valido salientar que pensando em engenharia de software que antes de qualquer código, geralmente documentamos o "porquê" de cada decisão.

Abaixo estão um quadro com todos os ADRs deste desafio. Cada um documenta o contexto, as opções consideradas, as decisões e justificativas, bem como as consequências esperadas.
